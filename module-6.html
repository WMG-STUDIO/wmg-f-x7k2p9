Module 6 · 60 minutes · prérequis : Module 5 réussi (première publication faite)

L'usine cloud : ton ordinateur peut dormir

Jusqu'ici, le facteur travaille quand TU lances la commande. Dans ce module, on déménage toute l'équipe chez GitHub : tous les 3 jours à 8h, le carrousel se prépare tout seul, ton téléphone sonne, tu réponds /publier — et c'est en ligne. Ordinateur éteint, en vacances, au ski : l'usine tourne.

Expliqué comme à un bébé — GitHub Actions
GitHub ne fait pas que stocker des dossiers : il prête gratuitement des ordinateurs qui exécutent tes instructions. Tu écris sur une feuille « tous les 3 jours à 8h, fais ceci » (cette feuille s'appelle un workflow), tu la ranges dans ton dépôt, et GitHub règle son réveil-matin. C'est notre déménagement : le dessinateur, le messager et le facteur quittent ton ordinateur pour l'immeuble GitHub, ouvert 24h/24.

6.1Créer le bâtiment de l'usine (dépôt privé)

1
Sur github.com : + → New repository → nom : mon-usine → visibilité : Private 🔒 (l'usine contient ta stratégie de contenu — elle ne regarde personne) → coche « Add a README » → Create.
création du dépôt privé mon-usine, case Private fléchée

6.2Déménager les fichiers (sans ligne de commande)

1
Dans ton dépôt mon-usine → Add file → Upload files.
2
Ouvre le dossier du kit sur ton Bureau, sélectionne TOUT SON CONTENU (Cmd+A / Ctrl+A) SAUF trousseau.env (tes clés ne montent JAMAIS ici — décoche-le / ne le glisse pas) → glisse tout dans la page GitHub.
Vérification vitale avant de valider
Dans la liste des fichiers à envoyer affichée par GitHub, vérifie que trousseau.env N'Y EST PAS. Le kit contient un garde-fou (fichier .gitignore) qui le bloque, mais on vérifie quand même à l'œil : c'est ton token.
3
Bouton vert Commit changes → patiente (il y a des images) → tes dossiers apparaissent : template/, publications/, robots/…
le dépôt rempli après upload, arborescence visible

6.3Le coffre-fort des clés (les « secrets »)

Sur ton ordinateur, les clés vivaient dans trousseau.env. Chez GitHub, elles vont dans un coffre chiffré que même toi ne pourras plus relire (juste remplacer) :

1
Dans mon-usine → Settings (onglet en haut) → menu gauche : Secrets and variables → Actions → bouton New repository secret.
chemin Settings → Secrets and variables → Actions
2
Crée les 3 premiers secrets, un par un — le NOM doit être EXACTEMENT celui-ci (majuscules comprises) :
· Name : IG_ACCESS_TOKEN → Secret : colle ton token
· Name : IG_USER_ID → Secret : tes 17 chiffres
· Name : GITHUB_IMAGES → Secret : ton-compte/mes-images

6.3 bis — Le 4ᵉ secret : le passe-étagère du dessinateur

Pourquoi un 4ᵉ secret ?
Au Module 5, c'est TOI qui déposais les images sur l'étagère mes-images. Maintenant c'est le robot dessinateur — il lui faut donc son propre passe pour y entrer. Chez GitHub, ce passe s'appelle un « jeton d'accès » : on va en fabriquer un limité uniquement à l'étagère (s'il fuyait, il n'ouvrirait rien d'autre).
a
Ton avatar (en haut à droite) → Settings → tout en bas à gauche Developer settings → Personal access tokens → Fine-grained tokens → Generate new token.
b
Name : passe-etagere · Expiration : 1 an (note un rappel dans ton agenda !) · Repository access : Only select repositories → mes-images · Permissions : Contents → Read and write → Generate token.
création du fine-grained token : accès limité à mes-images + permission Contents
c
Copie le passe affiché (il commence par github_pat_…) → retourne dans mon-usine → Settings → Secrets → New repository secret → Name : GH_IMAGES_TOKEN → colle → Add.
3
La liste affiche tes 4 secrets avec des pastilles vertes. GitHub ne montrera plus jamais leur valeur : c'est le principe du coffre.
liste des 4 secrets créés

6.4Réveiller le dessinateur (workflow « préparer »)

Les feuilles d'instructions des robots sont déjà dans le kit, dossier robots/ (chez GitHub, il s'affiche .github/workflows/). Il y en a deux :

RobotFeuilleCe qu'il fait
🎨 Le dessinateur-messagerpreparer.ymlTous les 3 jours à 8h : dessine le prochain carrousel, dépose les aperçus, t'ouvre un ticket de validation
📮 Le facteurpublier.ymlQuand tu écris /publier dans le ticket : publie tout (carrousel + quiz + story) et ferme le ticket
1
Onglet Actions de ton dépôt → si GitHub demande « I understand my workflows, go ahead and enable them » → clique (tu actives tes robots).
2
Test immédiat sans attendre 3 jours : dans Actions → menu gauche « Préparer le prochain carrousel » → bouton Run workflow → Run workflow. Le robot démarre (rond jaune → coche verte, ~2 minutes).
bouton Run workflow + le run qui passe au vert

6.5Ton téléphone sonne : le ticket de validation

1
📱 Ding ! L'app GitHub t'envoie une notification : « 📋 Validation — carrousel prêt ». Ouvre-la.
2
Le ticket (une « issue ») montre toutes les slides en aperçu, la story, et la légende prévue. Fais défiler, vérifie.
l'issue sur le téléphone : aperçus + légende, défilement
3
Tout te va ? En bas du ticket, écris le commentaire : /publier → envoie. (Un souci ? Écris /annuler : le carrousel est mis de côté, rien ne part.)
4
2 minutes plus tard, nouvelle notification : « ✅ Publié sur ton compte ! ». Ouvre Instagram : c'est en ligne. Tu n'as pas touché un ordinateur.
le commentaire /publier + la réponse ✅ du robot + le post en ligne (3 écrans)
Ce qui vient de se passer, en une phrase : un ordinateur gratuit de GitHub s'est allumé, a dessiné tes slides, t'a demandé la permission, a publié sur Instagram via ta clé officielle, puis s'est éteint. Il recommencera tous les 3 jours, pour toujours, sans toi.
Point de contrôle final du Module 6 — le test « ordinateur éteint »
  1. Mes 4 secrets sont dans le coffre (pastilles vertes) ✔
  2. Le run « Préparer » passe au vert ✔
  3. J'ai reçu la notification, validé avec /publier, et le post est en ligne — avec mon ordinateur ÉTEINT ✔
3/3 ? Ton usine est autonome. Éteins ton ordinateur ce soir la conscience tranquille : rendez-vous dans 3 jours sur ton téléphone. 🏭✨

Glossaire du module — expliqué comme à un bébé

Workflow 📋 — la feuille d'instructions d'un robot GitHub : « quand X arrive, fais Y ».

Cron ⏰ — le réveil-matin des robots. 0 6 */3 * * se lit « à 6h (temps universel = 8h chez nous l'été), tous les 3 jours ».

Secret 🔒 — une clé rangée dans le coffre chiffré de GitHub. Les robots peuvent s'en servir, personne ne peut la relire.

Issue / ticket 🎫 — une fiche de discussion dans un dépôt. Chez nous : le bon de validation que le messager t'apporte.

Run 🏃 — une exécution d'un robot. Vert = réussi, rouge = raté (et le Module 7 t'apprend à lire pourquoi).

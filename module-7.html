Module 7 · 30 minutes · le module que tu rouvriras souvent

Faire vivre ton usine

L'usine tourne. Ce dernier module t'apprend le quotidien du patron : nourrir la machine en contenu, lire son journal, réparer les 5 pannes classiques — et la seule action manuelle qui restera à jamais la tienne.

7.1Le journal de production : l'état de l'usine en un coup d'œil

Dans ton dépôt mon-usine, le fichier journal-production.md est tenu à jour par les robots eux-mêmes : chaque ligne dit si un carrousel est « en file d'attente », « en validation » ou « publié le… ». C'est la première chose à regarder quand tu te demandes « où en est-on ? ».

le journal dans GitHub avec les 3 statuts surlignés

7.2Nourrir l'usine : créer TES carrousels

Jusqu'ici tu as publié l'exemple du kit. Voici la boucle de production de TON contenu — 30 minutes par carrousel au début, 15 avec l'habitude :

1
Duplique le dossier publications/exemple-01 → renomme la copie publications/02-mon-sujet (toujours un numéro d'ordre + un mot).
2
Dans VS Code, ouvre chaque slide-XX.html de la copie et change uniquement les textes (repérables entre les balises, exactement comme dans un formulaire). Garde la structure : promesse → rappel → l'idée pas à pas → quiz → récap.
3
Mets à jour legende.txt (le texte du post + la ligne « Réponse au quiz : … »).
4
Vérifie en local : double-clique GENERER → regarde les PNG (un texte trop long se voit tout de suite : il déborde — raccourcis-le).
5
Téléverse le dossier dans mon-usine/publications/ (Add file → Upload files, comme au Module 6). C'est tout : le dessinateur le prendra automatiquement à son tour dans la file.
la file publications/ avec 3 dossiers numérotés en attente
L'astuce du dimanche soir
Prépare 2 ou 3 carrousels d'un coup, une fois par semaine. La file se remplit, l'usine pioche toute seule, et tu as toujours une semaine d'avance. Un assistant IA (ChatGPT, Claude…) peut t'aider à rédiger les textes des slides — donne-lui ta structure (promesse/rappel/idée/quiz/récap) et ton sujet, garde ton style.

Réflexe universel en cas de panne : rien de ce que tu peux cliquer ne casse définitivement quoi que ce soit. Le pire scénario possible = régénérer un token (5 minutes). Respire, lis le message d'erreur, cherche-le dans ce tableau.

7.4La seule action manuelle restante : épingler le quiz

L'API de Meta sait poster le commentaire-réponse du quiz, mais pas l'épingler (le fixer en haut). 10 secondes après chaque publication :

1
Ouvre ton post → le commentaire du quiz (posté par toi) → appui long dessus → Épingler. Fini.
appui long sur le commentaire + option Épingler

7.5La sécurité de ton usine (à relire 2 fois par an)

Les 4 commandements des clés
  1. Le token et le fichier trousseau.env ne se partagent jamais, avec personne, sous aucun prétexte (même « le support Instagram » — c'est TOUJOURS une arnaque).
  2. Les clés ne vont jamais dans le dépôt public mes-images.
  3. Un doute, une fuite ? Révoquer → regénérer → remplacer le secret (Module 3.6 + 6.3). 10 minutes, aucun dégât.
  4. La double authentification reste activée à vie sur Facebook ET GitHub.

7.6Et après ?

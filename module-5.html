Module 5 · 45 minutes · prérequis : le trésor du Module 3 (identifiant + token)

Ta première publication automatique 🎉

C'est le grand jour : à la fin de ce module, un carrousel complet — avec sa légende, son commentaire quiz et sa story — sera publié sur ton Instagram par ton ordinateur, pendant que tu regardes.

5.0Apprivoiser le terminal (10 minutes, une fois pour toutes)

Expliqué comme à un bébé
Le terminal, c'est parler à ton ordinateur par écrit au lieu de cliquer. Tu tapes une phrase (une « commande »), Entrée, il obéit et répond par écrit. C'est tout. Ça a l'air austère (fenêtre noire, curseur qui clignote) mais c'est juste une conversation — et tu n'auras que 3 phrases à connaître, toutes fournies à copier-coller.
Mac
Appuie sur Cmd + Espace (la loupe) → tape Terminal → Entrée. La fenêtre s'ouvre.
Win
Menu Démarrer → tape PowerShell → Entrée. La fenêtre bleue s'ouvre.
Terminal Mac et PowerShell Windows côte à côte, annotés « c'est juste une conversation »
1
Ta première phrase : tape echo bonjour puis Entrée. L'ordinateur répond bonjour. Voilà, tu sais parler terminal. 😄
Copier-coller dans le terminal : Mac = Cmd+C / Cmd+V comme partout. Windows PowerShell = clic droit pour coller. Toutes les commandes de la formation se copient depuis l'espace membre avec le bouton 📋 — tu ne taperas jamais rien à la main.

5.1Installer Python (le moteur du facteur)

Python ?
C'est le langage dans lequel le script du facteur est écrit. Tu n'apprendras pas Python — tu installes juste son « lecteur », comme on installe un lecteur vidéo pour lire un film.
Mac
Bonne nouvelle : déjà installé. Vérifie dans le Terminal : python3 --version → il répond « Python 3.x ». ✔
Win
Va sur python.org/downloads → bouton jaune « Download Python » → lance l'installateur → ⚠️ COCHE LA CASE « Add python.exe to PATH » en bas AVANT de cliquer Install. C'est l'erreur n°1 des débutants : sans cette case, l'ordinateur « ne trouve pas » Python ensuite.
installateur Windows avec la case Add to PATH entourée en rouge
✓
Vérification (les deux systèmes) : ferme et rouvre le terminal → python3 --version (Mac) ou python --version (Windows) → « Python 3.x » s'affiche. ✔

5.2Le trousseau de clés : le fichier .env

Le facteur a besoin de tes 2 trésors du Module 3. On les range dans un petit fichier que le script lira — et qui ne quittera jamais ton ordinateur.

1
Dans VS Code (dossier du kit ouvert), ouvre le fichier trousseau.env :
IG_ACCESS_TOKEN=COLLE_TON_TOKEN_ICI
IG_USER_ID=COLLE_TES_17_CHIFFRES_ICI
GITHUB_IMAGES=TON-COMPTE-GITHUB/mes-images
2
Remplace les 3 valeurs (token du Module 3.6, identifiant du Module 3.3, ton compte GitHub du Module 4). Pas d'espaces autour du =, pas de guillemets. Enregistre (Cmd/Ctrl+S).
Ce fichier est ton coffre
trousseau.env contient ton token. Il reste sur TON ordinateur : ne l'envoie jamais, ne le copie jamais dans le dépôt public mes-images. (Au Module 6, on utilisera le coffre-fort chiffré de GitHub à la place — encore plus sûr.)

5.3Ton premier carrousel : générer puis mettre en rayon

Le kit contient un carrousel d'exemple complet (publications/exemple-01/) : 8 slides + la story + le fichier legende.txt (le texte sous le post + la réponse du quiz). Pour aujourd'hui, on publie l'exemple tel quel — tu personnaliseras au Module 7.

1
Génère les images : double-clique GENERER-Mac.command / GENERER-Windows.bat → les PNG apparaissent dans publications/exemple-01/rendu/, déjà nommés exemple-01-slide-01.png, etc.

Mets-les en rayon : comme au Module 4, va dans ton dépôt mes-images → Add file → Upload files → glisse tout le contenu du dossier rendu/ → Commit changes. (Au Module 6, les robots feront cette étape à ta place — là, on apprend le geste une fois.)

5.4La répétition générale : le « dry-run »

Dry-run ?
« Répétition à sec » : le facteur fait TOUTE sa tournée (préparer les images, les déposer sur l'étagère, vérifier les accès Meta)… sauf le dernier geste. Rien n'apparaît sur Instagram. C'est le filet de sécurité avant chaque première fois.
1
Dans le terminal, colle la commande fournie :
python3 publier.py exemple-01 --repetition (Windows : python au lieu de python3)
2
Lis ce qu'il raconte : il vérifie ton trousseau ✓, puis chaque image de l'étagère une par une ✓✓✓, et conclut « 🟡 RÉPÉTITION terminée : tout est prêt, rien n'a été publié ». Si une image manque, il te dit laquelle et quoi faire.
sortie du dry-run complète, annotée ligne par ligne

5.5🚀 La vraie première fois

1
Même commande, sans le filet :
python3 publier.py exemple-01
2
Regarde le facteur travailler (30 à 60 secondes) : images → étagère → « containers » Meta → « ✅ Publié ! » → commentaire quiz posté → story publiée.
3
Ouvre Instagram sur ton téléphone. Ton carrousel est là. Ta story aussi. Tu viens de publier sans toucher Instagram. 🎉 (Le post peut mettre 1-2 minutes à apparaître : Meta « digère ». C'est normal.)
le terminal « ✅ Publié ! » + le post visible sur le téléphone, côte à côte
Si le facteur dit « Media ID is not available »
Traduction : « Meta n'a pas fini de digérer, patiente ». Le script réessaie tout seul jusqu'à 5 fois. Si ça échoue quand même : attends 2 minutes et relance la même commande. (C'est LE message le plus fréquent — et le plus inoffensif.)
Point de contrôle final du Module 5
  1. Le carrousel d'exemple est sur mon profil Instagram, publié par le script ✔
  2. Le commentaire quiz est sous le post, la story est en ligne ✔
  3. Je sais ce que fait --repetition et quand l'utiliser ✔
3/3 ? Ton facteur est opérationnel. Dernier chantier : le faire travailler tout seul, à heure fixe, ordinateur éteint. C'est le Module 6.

Glossaire du module — expliqué comme à un bébé

Terminal ⌨️ — parler à l'ordinateur par écrit. Une commande = une phrase + Entrée.

Python 🐍 — le langage du script. Tu installes son lecteur, pas besoin de l'apprendre.

Script 📜 — une liste d'instructions que l'ordinateur exécute dans l'ordre. Notre script s'appelle publier.py : c'est le cerveau du facteur.

.env 🔐 — le trousseau : un fichier local qui contient tes clés. Le script le lit, personne d'autre.

Container 📦 — le « colis en préparation » chez Meta : tu déposes les images, Meta les emballe, puis tu dis « envoie ! ». Le script gère ça tout seul.
